## What changed in v1.4 vs v1.3

### Code

- Deleted legacy non-agentic R1/R2/R3 runners
- Deleted experimental R4/R5 Stanford-Minion wrappers (and the `mcp` / `rank_bm25` / `requests` transitive deps they pulled in)
- Renamed `src/hybrid_coding_eval/runners/` → `src/hybrid_coding_eval/agents/` (mechanical rename to reflect what the directory contains after the cleanup)
- `bench sweep` now auto-spawns the router proxy, reading `models.local` from the config
- Added `--external-router` flag to `bench sweep` for users who want to manage the router themselves
- Bumped version to 1.4.0

### Docs

- README rewritten with the v1.4 hero, 4-command quickstart, 5 agents, 8 strategies, 3 local models
- `docs/REPRODUCING.md` rewritten for v1.4 with a new "How to read the results" section that maps each headline number to its exact `bootstrap_cis.json` cell key
- `docs/BENCHMARK_NEW_MODEL.md` rewritten for v1.4 (5 agents, 18 tasks, 8 strategies, gemma4 baseline)
- `AGENTS.md` updated for the post-cleanup `agents/` directory structure and the auto-spawn-router workflow

### Infrastructure

- `requirements.txt` now mirrors `pyproject.toml` — added `pydantic` and `pyyaml` (was pyproject-only, broke fresh installs)
- CI workflow simplified: dropped the `Clone vendored Minions` and `Install Minions transitive deps` steps (no longer needed without R4/R5)
